normal git flowing

  • very first, we check, if our working directory have a git repository with git log
  • if no git repo is found, then create a repo by git init
  • now there are three stages on git. (1) working directory. (2) staging area. (3) commit
  • for git to have track on the working directory files, we need to push our files to staging area by git add 'fileName'
  • after staging our working file, we will push this staged file to commit area. so that, git can keep tracks of all commited files. to commit file git commit -m "commit message"
  • now lets say, we want to have huge changes in our current working file or distributing works between team members. for that we need to have a branch!!! git branch [branch name]
  • now lets switch to another branch! to get all the branches from the repo, git branch and to switch another branch git switch [another branch name]
  • after working on new branch, lets say, we want to merge this new branch to our main branch. first switch to our main branch then git merge [new branch name with new content] - this process is known as fast-forward merge. to get all the info of merging, click here
