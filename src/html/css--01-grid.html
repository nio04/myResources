unit

fr

fr unit represents a fraction of the available space in the parent container

auto

width will only occupy its content size

keyword

information about track

              
                grid-template-columns: repeat(3, 1fr);

in css grid, track means a full column side or a full row side. in above example, there are 3 column track

repeat() 

              
                grid-template-columns: repeat(3, 1fr);

repeat 1fr in column side for 3 times

min-content

              
                grid-template-rows: repeat(3, min-content)

  • row flexible
  • makes line break
  • may overflow
  • when define in a template-row, the max size of element content of that row, become the actual row size

max-content

              
                  grid-template-columns: repeat(3, max-content);

  • column flexible
  • makes no line break
  • no overflow
  • when define in a template-column, the max size of element content of that column, become the actual column size

minmax()

              
                grid-template-columns: repeat(min-content, minmax(10rem, 1fr));

if the track size have enough space to fit its content, it will use the SECOND size. if the container size keep decreasing, then the track size will also keep decreasing, but it will never be less than the FIRST size

auto-fill

              
                  grid-template-columns: repeat(auto-fill, 5rem);

depends on the provided size, if the container have extra space, then more empty coulmns will be created to fill up the full width of the container. this makes the grid alignment management poor-er as empty columns fill the remaining space

auto-fit

              
                grid-template-columns: repeat(auto-fit, 5rem);

depends on the provided size, if the container have enough space, then more empty columns will be created to fill up the full width of the container (just like AUTO-FILL) but it will auto collapse the newly created columns as it was never created!

BEST for LAYOUT. because, hence its auto collapse the newly created coumns, the grid can have alignment properties effect

row side defining

              
                grid-template-rows: [define row size]

define how many row to be generated

naming template

              
                  grid-template-rows: [first-start] 1fr [first-end second-start] 10rem [second-end];

naming grid template rows

justify-items

              
                  justify-items : stretch[default] center | end | start;

make each cell a container. then depends on the provided property, it controls the element position in x axis (horizontal)

justify-content

              
                  justify-content : stretch | start | end | center | space-around | space-between |space-evenly;

when the total size of grid is less than the container size... then this property align the grid in main axis (horizontal)

column side defining

              
                grid-template-columns: [define column size];

define how many column to be generated

align-items

              
                  align-items : stretch [default] | start | end | center | baseline;

make each cell a container. then depends on the provided property, it controls the element position in y axis (verticle)

align-content

              
              align-content : stretch | start | end | center | space-between | space-around | space-evenly

when the total size of grid is less than the container size... then this property align the grid for cross axis (verticle)

other properties for parent container

gap

              
                  gap : [row gap] [Column gap];

shorthand property for row-gap and column gap. [column gap] define the gap between two column. same apply for row side

grid-auto-flow

              
                grid-auto-flow : row[default] | column | row dense | column dense

when set to COLUMN, implicit grid tracks will be added as column. then the cell order also get messed up

grid-auto-row

              
                  grid-auto-rows : [size];

define height for implicit tracks for row side

grid-auto-columns

              
                   grid-auto-columns : [size];

define width for implicit tracks for column side

naming grid lines

              
                .container{
                grid-template-rows: [header-start] 200px [header-end main-start] 1fr [main-end
                footer-start] 300px [footer-end]
                grid-template-columns: repeat(3, [col-start] 1fr [col-end]) 200px [grid-end]
                }

                .header{
                grid-column : col-start 1 / grid-end;
                grid-row: header-start / header-end;
                }
                .main{
                grid-column: col-start 1 / grid-end;
                grid-row: main-start / main-end;
                }
                .footer{
                grid-column: col-start 1 / grid-end;
                grid-row: footer-start / footer-end;
                }

only one thing to note, when we name grid line inside repeat(). with the naming, there will be a number by sequence. format (col-start 1 or col-start-5 or col-end 7)

naming grid area

              
                .container{
                grid-template-areas: "head head"
                                     "box  box  box  side"
                                     "main main main side"
                                     "main main main side"
                                     "foot foot foot foot";
                }

                .header{
                grid-area: head;
                }
                .box{
                grid-area: box;
                }
                .sidebar{
                grid-area: side;
                }
                . main{
                grid-area: main;
                }
                .footer{
                grid-area: foot;
                }

miscs

explaining useful template

              
                grid-template-columns: repeat(auto-fit, minmax(min-content, 25rem));

intially, if the parent container has enough space, then the columns will have 25rem. if the container size decrease & the column tracks can not have 25rem... it will apply min-content effect. which means, contents will have flexible row size

famous snippets

              
                grid-template-columns : repeat(auto-fit, minmax(250px, 1fr));

if the container size keeps decreasing as first column can not keep the cells in first column... then it creates a new row. it creates a new row, hence container size keeps decreasing and our content have minimum space specified.

since new rows are created, we can style those implicit rows by grid-auto-rows

famous snippets

              
                grid-template-columns : repeat(auto-fill, minmax(200px, 1fr));
