unit

fr

portion of the remaining space

keyword

repeat() 

              
                grid-template-column : repeat(3, 1fr);

repeat 1fr in row side for 3 times

min-content

              
                grid-template-row : repeat(3, min-content)

  • row flexible
  • makes line break
  • may overflow

max-content

              
                  grid-template-column : repeat(3, max-content);

  • column flexible
  • makes no line break
  • no overflow

minmax()

              
                grid-template-column : repeat(min-content, minmax(10rem, 1fr));

minmax(first, second) usually used in column side. if the container have enough space, children items will have [second] size. and when the parent container size keeps decreasing, the children items size will never be less than the [first] size.

auto-fill

              
                  grid-template-column : repeat(auto-fill, 5rem);

used in column side. depends on the provided size, if the container have extra space than more empty coulmns will be created to fill up the full width. i think, its best to ignore this property

auto-fit

              
                grid-template-column : repeat(auto-fit, 5rem);

used in column side. with provided size, the children items try to fit per row

                grid-template-row : [define row size]

naming template

              
                  grid-template-row : [first-start] 1fr [first-end second-start] 10rem [second-end];

naming grid template rows

justify-items

              
                  justify-items : stretch[default] center | end | start;

main axis

justify-content

              
                  justify-content : stretch | start | end | center | space-around | space-between |space-evenly;

when the total size of grid is less than the container size... then this this property works for main axis (x)

column side defining

              
                grid-template-column : [define column size];

define how many column needed

align-items

              
                  align-items : stretch [default] | start | end | center | baseline;

apply effect on cross axis / verticle side

align-content

              
              align-content : stretch | start | end | center | space-between | space-around | space-evenly

when the total size of grid is less than the container size... then this this property works for cross axis (y)

other properties for parent container

shorthand property for row-gap and column gap

                  grid-auto-rows : [size];

define height for implicit cell for row side

grid-auto-columns

              
                   grid-auto-columns : [size];

define width for implicit cell for column side

grid-auto-flow

              
                  grid-auto-flow : row[default] | column | row dense | column dense

shorthand for grid-auto-row property and grid-auto-columns property

miscs

explaining useful template

              
                grid-template-column : repeat(auto-fit, minmax(min-content, 25rem));

intially, if the parent container has enough space, the elements will defaultly have 25rem && as the container size decrease & the elements can not have 25rem... it will simply apply min-content which means, contents will have flexible row size
