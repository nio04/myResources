MERGING : fast-forward

fast-forward merge

              
                git switch master
                git merge [branch name]
first, we switched to the branch, we want to merged into then we give the branch name git merge [branch name] (other brach, which works we want to merge) this fast-forward merge term is used, when we want to merge our newly branch works file - to our base branch & the base branch does not have any new commit from the moment, newly branch were created

MERGING : non - fast-forward

non : fast-forward merge :: [no-conflict]

              
                git switch master
                git merge [branch name]
first, we switched to the branch, we want to merged into then we give the branch name git merge [branch name] (other brach, which works we want to merge) then git will auto prompt us to give a commit message name after the commit message has been given, then the merging operation will be successful when our old branch have some more commit, that our new branch doesn't have and then if we want to merge our new branch - to old branch :: its called not fast-forward merging operation

non : fast-forward merge :: [conflict]

              
                  git switch master
                  git merge [branch name]
                  //get an error
                  //manually fix it
                  //stage
                  //commit
                  //merge again
first, we switched to the branch, we want to merged into we give the merge command now git will show us a error of confliction we manually fix the error stage the file commit file merge file
