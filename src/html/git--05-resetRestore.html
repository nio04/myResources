time travel

time travel

              
                /////////travel back to PAST/////////
                git checkout [particular commit hash]
                or
                git restore --source [any commit hash] [file name]

                /////////travel back to preset/////////
                git switch [current branch name]
                or
                git restore [file name]

*AFTER TIME TRAVEL TO PAST, that particular commit hash will be our new HEAD. time travel doesn't make loose our works!

tips : travel back to a specific commit. create a new branch from there. now you can eliminate unnecessary/error works from your working branch. or merge changes from new branch to our working branch

git restore

unsatged :: last commit

              
                git restore [file name]
                or
                git checkout HEAD [file name]
                or
                git checkout -- [file name]

unstaged works revert back to the last commit

staged :: unstaged

              
                git restore --staged [file name]

to unsatged a staged file

git reset

git reset

              
               git reset [any commit hash]

keeps the changes in working directory but removes all the commit histories from the latest commit till the specific commit hash! NO GOING BACK!!! tip -- only use this delete some commit history from log tip -- if you dont want to keep deleting commits from the latest commit. then first, git checkout [commit hash] to any particular commit (then that commit will be our new -head). second, then do normal or --hard reset then, only few commits from the middle will be removed

git reset --hard

              
               git reset --hard [any commit hash]

removes the changes from working directory and removes the commit log

tip -- if you dont want to keep deleting commits from the latest commit. then first, git checkout [commit hash] to any particular commit (then that commit will be our new -head). second, then do normal or --hard reset then, only few commits from the middle will be removed

git revert

git revert

              
             git revert [any commit hash]

  • takes the unnecessary changes by commit hash
  • make a new auto commit
  • remove the unnecessary changes from workplace
  • keep the previous unnecessary/unwanted commit history/histories available

know that - git revert is notmally used when collaborating with multiple people
