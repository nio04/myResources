stash : operation

time travel

              
                /////////travel back to PAST/////////
                git checkout [particular commit hash]
                or
                git restore --source [any commit hash] [file name]

                /////////travel back to preset/////////
                git switch [current branch name]
                or
                git restore [file name]
*AFTER TIME TRAVEL TO PAST, that particular commit hash will be our new HEAD. time travel doesn't make loose our works!

tips : travel back to a specific commits. create a new branch from there :) now you can eliminate unnecessary works from your working branch

git restore

git restore for unstaged works

              
                git restore [file name]
                or
                git checkout HEAD [file name]
                or
                git checkout -- [file name]
unstaged works revert back to the last commit / HEAD

git restore to unstaged file

              
                git restore --staged [file name]
to unsatged a staged file, we provide --staged segement

git reset

git reset

              
               git reset [any commit hash]
keeps the changes but removes the commit history

tip -- now, you can move the unnecessary changes to another backup branch, if you need them

git reset --hard

              
               git reset --hard [any commit hash]
removes the changes and removes the commit log also

git revert

git revert

              
             git revert [any commit hash]
takes the unnecessary changes and make a new commit and remove the unnecessary changes from workplace but keep the previous unnecessary commit history/histories available

note -- it may throw a confliction error while merging. since, we removed some content

know that - git revert is notmally used when collaborating with multiple people
