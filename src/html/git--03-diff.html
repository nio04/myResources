diff : information

diff : important information

by git diff command, we can see various types of differences from our working file from different timeline.

---- old file content [file name = a]
++++ new file content [file name = b]

diff : operation

unstaged :: last commit

              
                git diff

compare codes from last commit to current unstaged works

staged :: last commit

              
                git diff HEAD

compare codes from last commit to current staged and unstaged works

see changes in a specific file

              
                git diff HEAD [file path/file name]

comapre code from last commit to staged and unstaged area for a specific file

diff for branches

              
                git diff [first branch name..second branch name]

ORDER MATTERS!!! [only works for commited changes]
if conflict exist, [first branch] content will have (-) sign && [second branch] changes will have (+) sign

if no conflict exist, content from second branch which does not exist in first branch will have (+) sign. meaning, we dont have these (+) sign contents

diff for commit hash

              
              git diff [first commit hash..second commit hash]

ORDER MATTERS!!! [first commit] will be like a book, where we will compare our changes to [second commit hash] if some changes does not exist in [second commit] comparing from [first commit] they will be shown as (-) sign if some content doe not exist in [first commit] comparing from [second commit] they will be shown as (+) sign
