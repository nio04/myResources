stash : information
stash changes : important information git stash hide our works from staged & unstaged area. so that we can switch to another branch, without having to worry about, conflict warning or unstaged works follow to another branch when we add works in a branch and if we switch to another branch, before commit. two things will happen if no conflict was found :: the new works will be follow to the switched branch if conflict was found :: git wont let me switch branch
stash : operation
no conflict 
              
                // to stash changes
                git stash
                // to un-stash changes
                git stash pop
if there were no conflict and unstaged works exist. then when we switch branch, our unstaged works follow the branch. to prevent this behavior, we stash the changes note :: the following works on branch is not permanent. when we commit the works from our working branch, all the follow up works will be gone!!!
conflict 
              
                // to stash changes
                git stash
                // to un-stash changes
                git stash pop
we don't want to commit our current works, but wanting to switch branch. and if there is conflict warning, then we stash the changes, to bypass the warning
